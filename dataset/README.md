# Dataset

## Provenance

All 45 `(incoming_email, sent_reply)` pairs in `emails.jsonl` are
synthetically generated by prompting a Groq-hosted LLM
(`llama-3.3-70b-versatile` by default), once per category/tone combination —
9 combinations, 5 pairs each. The generation timestamp and model are
recorded per row in `metadata`.

## Categories and Tones

- **Categories:** `customer_support`, `scheduling`, `sales`, `complaint`,
  `internal`.
- **Tones:** `formal`, `casual`, `apologetic`.

Not every category/tone pairing is represented; 9 of the 15 possible
combinations were selected for variety.

## Representativeness and Known Limits

**Synthetic data.** The dataset is not drawn from real correspondence. The
judge (`evaluator/`) runs on a different Groq model than the one used to
generate the dataset; the reply generator, however, uses the same model as
dataset generation (`GEN_MODEL`, `llama-3.3-70b-versatile` by default — see
`generator/config.py` and `dataset/build_dataset.py`), so only the judge is
independent. A sample of rows was also manually reviewed for realism.

**Single-turn only.** Each row is one incoming email and one reply — no
multi-message threads or back-and-forth negotiation.

**Register.** All categories are business-email scenarios, so the dataset
skews toward a professional register.

**Near-duplicate scenarios.** A few incoming-email subjects recur across
different category/tone combinations, since each combination is generated
independently. This extends beyond subject-line overlap: the "Issue with
recent order" cluster (`email-001`, `email-006`, `email-031`, `email-036`,
spanning `customer_support` and `complaint`) reuses the same customer name
and order number across all four rows. A second cluster ("Meeting Request
for Project Discussion", `email-011` and `email-016`, both `scheduling`)
repeats its subject with different specifics. Wording still varies row to
row, and all 45 `sent_reply` values are textually unique, but a few
scenarios are effectively duplicated rather than independent. This was
identified by manual spot-check, not an exhaustive audit.

**Scale.** 45 pairs give the retrieval step meaningful per-category variety
and the evaluator a reasonable sample for aggregate and per-category
scoring, but do not represent broad coverage of real-world email style.

## Regenerating

```bash
python -m dataset.build_dataset
```

Requires `GROQ_API_KEY`. Since generation is a live LLM call, re-running
will not reproduce the file byte-for-byte, but will produce a
comparably-sized dataset across the same category/tone combinations.
